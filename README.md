# ⚔️ MyDota (GAS & Multiplayer)

Привет! Это мой амбициозный пет-проект: попытка перенести ключевые механики Dota 2 на рельсы Unreal Engine 5, используя мощь C++ и Gameplay Ability System.   
Цель — создать расширяемый и производительный мультиплеерный фреймворк для MOBA-игр.

## 🏗 Архитектура систем (Мои мысли)
При разработке я буду придерживаться нескольких принципов, чтобы код не превратился в «спагетти»:
1. Весь базовый код будет на С++ классах, а настройки вынесены в BP, DataAsset, Config и тд для удобства быстрых обновлений без участия программиста
2. **Network Relevancy**: В MOBA важно, чтобы клиент не знал того, чего не видит его герой. Поэтму хочу, чтобы данные о врагах в «тумане» вообще не реплицировались.
3. **Component-Based Design**: разделение на компонентам, для упрощения  отладки, 

## 🚀 Что уже под капотом
На данный момент реализован фундамент:  
- [Fog Of War](Docs/FogOfWar.md) (Туман войны): Кастомная система видимости, учитывающая высоты и радиус обзора юнитов.
- Игровой цикл
- Интегрирован GAS (В процессе): Реализован в режиме прототипа, для проверки нанесения урона. Подключен ближний и дальний бой

## 🗺 Дорожная карта (Roadmap)
Вот что в планах на ближайшее время:

### Фаза 1: Статы
- Атрибуты Героя (Сила, ловкость, интеллект, скорость передвижения и тд)
- Атрибуты игрока (Золото)
- Расчеты нанесения урона, уменьшение урона

### Фаза 2: Крипы и Лайны
- Система спавна крипов по таймеру.
- ИИ для движения по вейпоинтам и поиска ближайшей цели (Behavior Trees).
- Реализация механики Gold/XP за ластхит.

### Фаза 3: Сражение и взаимодействия
- Магазин.
- Активные предметы и полноценный инвентарь.
- Механика стаков и баффов (Stun, Silence, Slow) через Gameplay Tags.

### Фаза 4: Полировка
- Оптимизация сетевого трафика.
- Оптимизация узких мест

## 🛠 Технологии
**Engine**: Unreal Engine 5.6  
**Language**: C++  
**Framework**: Gameplay Ability System (GAS)  
**Networking**: Client-Server (Authoritative) via **Dedicated Server**.
