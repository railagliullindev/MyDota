<a name="description"></a>
## Описание
Этот класс реализует систему тумана войны в стиле **Dota 2**.
Создаются два отдельных экземпляра (для Radiant и Dire), каждый управляет
видимостью для своей команды. 
Реплецируется только своей команде, менеджер другой команды у клиента не существует. 

---

<a name="architecture"></a>
## Архитектура
- **Сервер**: рассчитывает видимость, сжимает данные в битовый массив
- **Клиент**: получает сжатые данные, обновляет текстуру для пост-процесс эффекта

---

<a name="features"></a>
## Фичи
- Система высот (High Ground) - возвышенности блокируют обзор
- Статические препятствия (деревья/стены) с тегом "BlockFog"
- Плавное затухание тумана на клиенте
- Оптимизация через кеширование видимых ячеек
- Сжатие данных для репликации (1 бит на ячейку)

### Зрение
Это самая важная часть для тумана войны и тут не обойтись комплесным решением

<a name="vision-calc"></a>
#### Вычисления
1. Для начала нам необходимы источники, кто будет делиться зрением, для этого нам необходимо зарегистрировать участника
2. Для расчета что видит источник на нужно:
   - Собрать ячейки в радиусе  
      [GIF рабочее пространство]()

   - Для провери используем лучи Брезенхема    
     [GIF лучи Брезенхема]()  
     [GIF Наши лучи]()  

   - Делаем выборку ячеек по следующим критериям:
     - Отсекаем ячейки со статическим объектом, причем эту ячейку мы видим
     - Отсекаем ячейки, которые находятся выше юнита источника  
     [GIF Результат отсеивания]()  
   
3. Записать вычисления

---

<a name="vision-optimization"></a>
#### Оптимизация
- Вместо того что бы пересылать всю картру, добавляем следующее:
  - Создаем кэш предыдущих вычислений
  - Добавляем проверку, если юнит не сдвинулся со своего тайла, то вычисления не требуются 
    - **Например**: Статичесике объекты - Башни, варды. И Динамические - крипы/герои, пока стоят на месте или деруться    
    [GIF Когда мы делаем расчеты?]()  
  - Если проверка необходима, Обнуляем закэшированное вычисление, и вычисляем для новой области, по его текущему положения и радиусу
  - Отправлять клиентам **только** изменившийся кусок  
    [GIF Клиент когда срабатывает OnRep_CompressedFog]()  

В итоге мы не проверяем в каждом кадре всю карту, а только ту часть где проихошли изменения, а так же сэкономили количетсво репликаций

Принудительно помечаем массив для репликации, так как TArray внутри не всегда триггерит репликацию при изменении элементов    
Это заставляет Unreal Engine реплицировать данные тумана только тогда, когда они реально изменились, экономя трафик в моменты затишья.

<a name="compress"></a>
### Сжатие

Для экономии трафика используем битовое сжатие для отправки его клиентам.  
    
- Берем наш `RawVisibilityData` и сжимаем его в `CompressedFogData` в итоге данные сжимаются в 8 раз.   
        
  - Вместо того чтобы слать массив байтов (где 1 ячейка = 1 байт), мы упаковываем данные в битмап. 
  - Одна ячейка — это всего 1 бит (0 или 1). (Пакет для сетки 256x256 весит всего около 8 КБ)
    
- Клиенты полученные данные разворачивают их в `PixelBuffer`
    - А дальше создают Region, и обновляют часть текстуры FogOfWar, который в свою очередь уже используется PP в мире (Что бы затемнить мир) и на миникарте
